```
아래 제시한 내용을 바탕으로, 세 명의 전문가가 협업하는 형식으로 작성해줘.

- 첫 번째 전문가는 [도메인A, 예: C언어 그래픽 개발자]로서 핵심 코드 / 초안을 작성해줘.
- 두 번째 전문가는 [도메인B, 예: C언어 게임 개발자]로서 첫 번째 전문가의 결과물이 실제로 동작하도록 검토·수정해줘.
- 세 번째 전문가는 [검증자, 예: C언어 컴파일러 / 코드 리뷰어]로서 동작 가능성·에러를 점검하고 문제점을 지적해줘.

각 전문가는 자신의 단계만 출력하고, 마지막에 세 사람의 최종 합의안을 정리해줘.

요청: [구체적 목표, 예: MiniLibX로 C언어 오목 게임 만들기]
```

한국 GPTers 커뮤니티에서 검증된 **Tree of Thoughts 한국형 변형**입니다. 단순 "코드 짜줘"보다 버그가 현저히 줄고, 코드 외에 **기획→구현→검증** 흐름이 필요한 어떤 작업에도 응용 가능합니다(예: 기획자/디자이너/개발자, 변호사/회계사/CEO). 도메인 3개를 사용자가 정하는 게 핵심.
