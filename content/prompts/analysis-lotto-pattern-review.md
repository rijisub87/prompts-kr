```
역할: 너는 데이터를 중립적으로 정리하는 분석가다. 패턴을 "설명"할 뿐, 미래를 "예측"하지 않는다.

입력 데이터(회차별 당첨번호 6개 + 보너스):
[PAST_RESULTS]

분석 기간: [PERIOD]

다음 항목을 표·수치로 정리:
1. 번호별 출현 빈도 (1~45, 많이/적게 나온 상위·하위 5개)
2. 홀짝 비율 분포 (6:0 ~ 0:6 중 어느 비율이 흔했나)
3. 고저 분포 (1~22 저구간 / 23~45 고구간)
4. 당첨번호 합계 분포 (최소·최대·평균·자주 나온 구간)
5. 연속수 등장 빈도 (예: 14-15처럼 붙은 번호)
6. 끝수(일의 자리) 분포

[제약]
- 모든 수치는 입력 데이터 범위 안에서만 계산, 없는 데이터 지어내지 말 것
- "그래서 다음엔 이 번호가 나온다"는 식의 예측·추천 절대 금지
- 과거 빈도는 미래 확률에 영향을 주지 않음(독립 시행)을 분석 끝에 1줄로 명시
- 표 위주로, 해석은 "관찰된 사실"까지만
```

로또는 매회 독립 시행이라 "덜 나온 번호가 나올 때가 됐다"(도박사의 오류)는 통계적으로 틀립니다. 이 프롬프트는 과거 데이터를 깔끔하게 정리하는 회고용이며, 미래 예측 도구가 아닙니다. 오히려 도박사의 오류를 직접 반박해 통계 교양에 가깝습니다.
